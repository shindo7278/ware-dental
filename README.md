# Hullbridge Dental Clinic — موقع كامل (Next.js)

هيكلة Next.js 15 رسمية وكاملة، جاهزة للتشغيل بـ `npm run dev`.

---

## خطوات التشغيل بالترتيب (المرة الأولى)

### 1. تثبيت الباكدجات
```bash
cd hullbridge-nextjs
npm install
```

### 2. تجهيز قاعدة البيانات
- اعملي حساب مجاني على [supabase.com](https://supabase.com) أو [neon.tech](https://neon.tech)
- اعملي مشروع جديد، وخدي الـ **Connection String** بتاعه

### 3. إعداد ملف البيئة
```bash
cp .env.example .env
```
افتحي `.env` واملي على الأقل:
- `DATABASE_URL` — الرابط اللي خدتيه من Supabase/Neon
- `ADMIN_JWT_SECRET` — أي نص عشوائي طويل (تقدري تولّديه بالأمر: `openssl rand -base64 32`)

> `RESEND_API_KEY` ممكن تستناها — الموقع هيشتغل من غيرها، بس إشعار الإيميل بتاع فورم Contact مش هيتبعت لحد ما تحطيها.

### 4. إنشاء الجداول في قاعدة البيانات
```bash
npm run prisma:migrate
```
هيسألك على اسم للـ migration، اكتبي أي اسم زي `init` واضغطي Enter.

### 5. عمل أول حساب أدمن
```bash
npm run seed:admin
```
ده هيعمل حساب بإيميل `admin@hullbridgedentalclinic.co.uk` وباسورد `AaMm778899$$` (تقدري تغيّريهم في `.env` قبل التشغيل لو حبيتي).

### 6. تشغيل الموقع محلياً
```bash
npm run dev
```
افتحي المتصفح على: **http://localhost:3000**

لوحة الأدمن: **http://localhost:3000/admin/login**

## هيكل المشروع

```
app/
├── page.js                 الصفحة الرئيسية
├── services/                فهرس الخدمات + [slug] لكل خدمة
├── our-prices/               صفحة الأسعار
├── blog/                     فهرس المدونة + [slug] لكل مقال
├── contact/                   صفحة التواصل
├── book/                       صفحة الحجز أونلاين
├── admin/                      لوحة التحكم (محمية بـ middleware.js)
│   ├── login/                  تسجيل الدخول
│   ├── bookings/                إدارة الحجوزات
│   ├── pricing/                  إدارة الأسعار + الخدمات المترابطة
│   ├── opening-hours/             مواعيد العمل
│   ├── homepage/                   فيديو الصفحة الرئيسية
│   └── messages/                    رسائل Contact Us
└── api/                          كل الـ API endpoints

components/                       مكونات React مشتركة (هيدر، فوتر، فورمات)
lib/                                Prisma client + التحقق من جلسة الأدمن
prisma/                              schema.prisma + سكريبت seed-admin
middleware.js                         يحمي كل صفحات /admin تلقائياً
```

## ⚠️ قبل النشر الفعلي (مش لازمة للتجربة المحلية)

1. **اللوجو**: حطي الملف في `public/images/logo.png` — هيظهر تلقائياً في الهيدر
2. **Google Business Profile**: تأكدي إنه موجود ومتوثق قبل النشر، عشان التقييمات والخريطة يشتغلوا بشكل كامل
3. **Resend API key**: لتفعيل إشعار الإيميل الفوري لرسائل Contact Us
4. **النشر**: أسهل طريقة هي رفع المشروع على [vercel.com](https://vercel.com) (مجاني للبداية)، وربط نفس قاعدة البيانات، وإضافة نفس متغيرات `.env` في إعدادات Vercel

## ملاحظات مهمة

- **الباسورد متشفّر**: حساب الأدمن الافتراضي بياخد الباسورد من `.env` وقت تشغيل `npm run seed:admin` بس، وبيتشفّر بـ bcrypt جوه قاعدة البيانات — مش متخزن كنص واضح في أي ملف كود.
- **محتوى الخدمات/المدونة الحالي placeholder**: الصفحات شغالة بمحتوى تجريبي مكتوب مباشرة في الكود (`SERVICES_DATA`, `POSTS_DATA`) كنقطة بداية. الخطوة التالية الطبيعية هي ربطها بجداول `Service`/`BlogPost` في قاعدة البيانات بدل الكود الثابت، عشان تتقدري تضيفي/تعدّلي من الأدمن مباشرة.
- **`previews/` (لو موجودة من نسخة سابقة)**: أي ملفات بهذا الاسم كانت للمراجعة التفاعلية في الشات بس، مش جزء من هذا المشروع الرسمي.
